# Glossary

This glossary defines recurring project terms so planning, code, UI, and documentation use the same language.

## Account

The local Maiks.yt user account. It can have multiple linked external accounts.

## Action Item

Something that needs review, approval, rejection, or deferral in the action panel.

Examples: price change approval, donation revocation request, withdrawal review, project archive summary.

## Action Panel

The review inbox for decisions that should not happen silently. Different from the live overlay control panel.

## Allocation

The assignment of money, credits, or platform-derived support to a project, item, stream goal, or other target.

## Channel

A themed public surface for a game, hobby, or creator persona.

Examples: Satisfactory, Programming, Tech, Minecraft.

## Credit

A site-managed value record that may be usable for future support allocation. Some credits may be payout-eligible, while restricted credits are not.

## Direct Donation

A payment made directly through the website/payment provider rather than through Twitch, YouTube, Patreon, or another platform.

## Event

A typed message representing something that happened.

Examples: chat message, donation received, stream goal updated, overlay notification triggered, AI readout requested.

## Guest

A person using the site without being signed in.

## Item

A specific thing inside a project.

Examples: GPU, subscription, clothing item, server part, wishlist entry, task-linked purchase.

## Ledger

The auditable record of money, credits, allocations, reappropriations, withdrawals, refunds, and spending.

Ledger entries should not be silently edited.

## Linked Account

An external account connected to a Maiks.yt account.

Examples: Discord, Twitch, YouTube, Patreon, Steam, Xbox.

## Milestone

A progress marker inside a project. Milestones can exist whether or not the project involves money.

## Overlay

An OBS browser-source surface shown on stream.

## Overlay Control Panel

The live stream control UI for changing overlay state, triggering test events, toggling chat, muting AI, switching layouts, and using emergency clean mode.

## Platform-derived Support

Support that came through an external platform and may need conversion or restrictions.

Examples: Twitch Bits, Twitch subs, YouTube memberships, Patreon support.

## Project

A container for work, funding, milestones, items, updates, or stream focus.

Projects can be monetary or non-monetary.

## Project Category

Why a project exists.

Examples: personal, family, content improvement, stream infrastructure, hobby, community.

## Project Type

The shape of a project.

Examples: one-time purchase, multi-item build, subscription, milestone-only, ongoing cost.

## Restricted Credit

A credit that can be used inside the site but cannot be paid out as cash.

Example: credit derived from Twitch subs or gifted subs.

## Stream Goal

The active project or item being tracked during a live stream. It can receive direct donations or default platform-derived allocations during that stream.

## Stream Session

A specific live stream occurrence. It may be linked to a scheduled stream, stream goal, projects, overlays, chat events, and simulator recordings.

## Theme

A visual identity for a channel, game, hobby, or stream mode. Overlay themes should be CSS-driven with a typed theme contract.

## Transport

The realtime communication method used between backend, overlays, and control panel.

Examples: WebSocket, SSE, local fake transport.

## Wishlist Entry

An item someone could buy or support. It can be internal or linked to external providers like Amazon or store wishlists.
